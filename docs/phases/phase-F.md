# Phase F — Stock & budget

## Goal

Between days, manage IT budget and order peripherals (toner, etc.). Shortages can leave someone unable to print (or similar) the next day. Leftover “waiting on order” state can carry into the next morning.

## In scope

- Between-day shop / order UI
- Budget currency from performance
- Stock inventory affecting ticket solvability
- Optional: day start with leftover blocked tickets

## Out of scope

- Full accounting sim depth

## Acceptance criteria

- [ ] Player can spend budget on at least two item types
- [ ] Zero stock blocks the matching fix until ordered
- [ ] Orders arrive on a clear rule (next day, etc.)
